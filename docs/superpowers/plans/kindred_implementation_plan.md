# Kindred Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan phase-by-phase. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the Kindred Flutter app — Google sign-in, profile onboarding, discover feed, likes, mutual-match chat — targeting Android with package `dev.mathminds.silver_fun`.

**Architecture:** Feature-first folders under `lib/features/`; shared code in `lib/core/`. Riverpod for state, go_router for navigation, repositories encapsulate all Firebase calls.

**Tech Stack:** Flutter 3.x · firebase_core/auth/firestore/storage · flutter_riverpod · go_router · google_sign_in · image_picker · flutter_image_compress · cached_network_image · google_fonts

**Design spec:** `docs/superpowers/specs/2026-04-29-kindred-design.md`

## Phase 4 — Feed + Likes

**Goal:** Discover feed streaming real Firestore profiles; heart-toggle likes with mutual match detection.

**Files to create/edit:**

| File | Purpose |
|---|---|
| `lib/features/feed/repository/feed_repository.dart` | `watchFeed(currentUid)` stream; `getUser(uid)` |
| `lib/features/feed/repository/likes_repository.dart` | `like`, `unlike`, `watchLikedByMe`, `watchLikedByOthers`; uses batch write to maintain reverse index `likedBy/{toUid}/from/{fromUid}` |
| `lib/features/feed/providers/feed_provider.dart` | `feedProvider` — stream of profiles with `.liked` merged from `likesProvider` |
| `lib/features/feed/providers/likes_provider.dart` | `LikesNotifier` — holds `Set<String>` of liked UIDs; `toggleLike(targetUid)` with match detection |
| `lib/features/feed/screens/feed_screen.dart` | 2-column grid; heart on each card; tap → `/profile/:userId` |
| `lib/features/feed/screens/profile_view_screen.dart` | Full-screen; hero photo; bio; chips; Like/Liked button |
| `test/features/feed/feed_repository_test.dart` | Excludes self + paused + unpublished |
| `test/features/feed/likes_repository_test.dart` | Like writes both collections; unlike removes both |

**Key data model note:** `LikesRepository.like(fromUid, toUid)` uses a Firestore batch to write `likes/{fromUid}/liked/{toUid}` **and** `likedBy/{toUid}/from/{fromUid}` atomically. `unlike` deletes both. This allows efficient "who liked me" queries without full-table scans.

**Steps:**
- [ ] Create `FeedRepository`: `watchFeed` queries `users` where `published==true && profilePaused==false`, excludes `currentUid`
- [ ] Create `LikesRepository`: `like`/`unlike` use batch writes; `watchLikedByMe(uid)` streams `likes/{uid}/liked/*` doc IDs as `Stream<Set<String>>`; `watchLikedByOthers(uid)` streams `likedBy/{uid}/from/*` doc IDs
- [ ] Create `likesProvider` as `StreamProvider<Set<String>>` wrapping `watchLikedByMe`; create `LikesNotifier extends AsyncNotifier` with `toggleLike(targetUid)` — writes like/unlike, then checks reverse index for mutual match, sets toast via `toastProvider`
- [ ] Create `feedProvider` as `StreamProvider` combining `feedRepository.watchFeed` with liked-set overlay (profile.liked = likedUids.contains(uid))
- [ ] Create `FeedScreen`: `GridView.builder` 2 columns; each card 3:4 photo + first name + age + city + 2 chips + `HeartButton`; tap card → `context.push('/profile/$userId')`
- [ ] Create `ProfileViewScreen(userId)`: loads user from `feedProvider` or `feedRepository.getUser`; square photo; scrollable body; fixed bottom bar with Back + Like buttons
- [ ] Wire `toastProvider` as `StateProvider<String?>` in `lib/core/providers/toast_provider.dart`; `toggleLike` sets toast "Liked {name}" or "It's a match with {name}! 🎉" then schedules clear in 2.2s
- [ ] Write repository tests
- [ ] Run `flutter test test/features/feed/` — pass
- [ ] Manual: open app (use second test account or seed data in Firestore console) → feed shows profiles → heart toggles → match toast fires
- [ ] Commit: `feat: feed and likes`

**Acceptance criteria:**
- Feed streams live from Firestore; own profile excluded
- Heart state persists across restarts
- Liking someone who liked you back → "It's a match!" toast
- `likedBy` reverse index written correctly (verify in Firestore console)

## Phase 6 — Chat

**Goal:** Mutual-match chat list and real-time message threads.

**Files to create/edit:**

| File | Purpose |
|---|---|
| `lib/features/chat/repository/chat_repository.dart` | `watchThreads(myUid)`, `watchMessages(chatId)`, `sendMessage`, `markRead`, `chatId(u1, u2)` helper |
| `lib/features/chat/providers/chats_provider.dart` | `matchesProvider` (mutual UIDs); `matchThreadsProvider` (list of `MatchThread`); `chatMessagesProvider(chatId)` |
| `lib/features/chat/screens/chat_list_screen.dart` | Match bubbles row + conversation list + unread badges |
| `lib/features/chat/screens/chat_screen.dart` | Bubble UI; match card; send bar; auto-scroll; marks read on open |
| `test/features/chat/chat_repository_test.dart` | Send + watch messages |

**Key details:**
- `chatId(uid1, uid2)`: `[uid1, uid2]..sort()` → join with `_`
- `matchesProvider`: intersects `watchLikedByMe` set with `watchLikedByOthers` set → `Stream<Set<String>>`
- `matchThreadsProvider`: for each match UID, fetches `UserProfile` and last message; emits `List<MatchThread>`
- `MatchThread`: `{ UserProfile user, ChatMessage? lastMessage, int unreadCount }`
- Chat messages are in `chats/{chatId}/messages` ordered by `sentAt asc`
- `markRead`: batch-updates all messages where `senderId != myUid && read == false` to `read: true`

**Steps:**
- [ ] Create `ChatRepository` with `chatId` helper, `watchMessages`, `sendMessage` (sets `read: false`), `markRead`
- [ ] Create `matchesProvider` combining liked-by-me and liked-by-others streams
- [ ] Create `matchThreadsProvider` as `StreamProvider<List<MatchThread>>` — joins matches with profiles and last message
- [ ] Create `chatMessagesProvider(String chatId)` as family `StreamProvider`
- [ ] Create `ChatListScreen`: horizontal `SingleChildScrollView` of match avatar bubbles; `ListView` of conversation rows with unread badge; empty state if no matches
- [ ] Create `ChatScreen(userId)`: header with back + avatar + name + heart; match card at top; `ListView` of bubbles (outgoing coral, incoming white); `TextField` send bar; send on tap or Enter; auto-scrolls to bottom; calls `markRead` on open
- [ ] Wire `/app/chats` and `/chat/:userId` routes
- [ ] Write `ChatRepository` tests with `FakeFirebaseFirestore`
- [ ] Run `flutter test test/features/chat/` — pass
- [ ] Manual: like Maya (a pre-seeded account or test account) who has liked you back → Chats tab shows match → tap → send message → appears in real time
- [ ] Commit: `feat: chat`

**Acceptance criteria:**
- Chats tab only shows mutual matches
- Messages persist in Firestore and stream in real time
- Unread badges update on receive; clear on open
