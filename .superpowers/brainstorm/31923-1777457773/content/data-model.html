Design — Section 2 of 3: Firebase Data Model

Firestore collections, document shapes, and security rules overview.

Firestore Collections

📂 users/{uid}
name: "Alex"
age: 27
bio: "..."
photoUrl: "gs://..."
interests: ["Coffee", "Jazz"]
published: false
profilePaused: false
createdAt: Timestamp
Writable only by owner. Readable by any authenticated user when published=true.
📂 likes/{uid}/liked/{targetUid}
likedAt: Timestamp
Writable by owner. To check if mutual: query likes/{targetUid}/liked/{uid}. A match exists when both docs exist.
📂 chats/{chatId}/messages/{msgId}
// chatId = sorted(uid1, uid2).join('_')
text: "Hey!"
senderId: "uid123"
sentAt: Timestamp
read: false
Readable/writable only by the two participants. Chat access requires mutual like to exist.
🪣 Firebase Storage
profile-photos/{uid}.jpg
Writable by owner, readable by any authenticated user. Max 5MB. Compressed to 800×800 before upload.

Security Rules Summary

✅ users/{uid} — read if published == true OR owner; write if owner only
✅ likes/{uid}/liked/{target} — write if request.auth.uid == uid; read if authenticated
✅ chats/{chatId}/messages — read/write only if auth.uid is one of the two participants AND mutual like exists
⚠️ Profile photo upload — max 5MB enforced in Storage rules; client compresses before upload as a UX improvement (not a security guarantee)

Photo Upload Flow

User picks image → Compress to 800×800 JPEG → Upload to Storage → Get download URL → Save to users/{uid}.photoUrl
